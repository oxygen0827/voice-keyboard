## 一、一句话定义

**Voice Keyboard 是一个软硬件结合的 AI 语音打字工具——按住按键说话，松手后文字自动出现在任意输入框里。**

不是"先打开一个语音输入面板→识别完→复制粘贴"。是真正意义上的"在哪都能说、说完直接打字"，就像键盘的延伸。

## 四、技术亮点（供对外宣传用）

### 4.1 "真正绕过输入法"的打字技术

大多数语音输入工具的问题是：文字会被输入法拦截，出现中英文乱码、繁简体转换等诡异问题。

Voice Keyboard 在三平台上用不同方案彻底绕过输入法：

- **macOS**：直接调用 Quartz `CGEventKeyboardSetUnicodeString` 向系统注入 Unicode 字符事件
- **Windows**：`SendInput + KEYEVENTF_UNICODE` 逐字发送，对微信/钉钉等 Electron 应用自动切换剪贴板粘贴模式
- **Linux**：X11 XTest 协议逐字输入

结果：无论什么应用（微信、钉钉、VS Code、浏览器、Word），文字都干净利落地出现，完全不经过 IME（输入法）。

### 4.2 五大 STT 引擎可选

| 引擎 | 延迟 | 中文能力 | 适合场景 |
|------|------|----------|---------|
| 讯飞语音听写 | 200-400ms | ★★★★ 逐字原文，数字自动转阿拉伯数字 | 中文日常推荐 |
| 阿里云 NLS | 200-400ms | ★★★★ 支持方言 | 方言用户 |
| 火山引擎 | 200-400ms | ★★★ 中文优化 | 字节生态用户 |
| 智谱 AI GLM-4-Voice | 500-800ms | ★★ 对话式改写 | 已有智谱 Key 的用户 |
| OpenAI Whisper | 500-1000ms | ★★ 多语言 | 英文/国际场景 |

推荐讯飞或阿里云——它们是专业 ASR 引擎，逐字原样转写，不乱加东西。

### 4.3 LLM 语音编辑 —— 这才是"生产力"

打完一句话后，按住编辑键说修改指令，LLM 自动执行。这是不是普通的"退格重打"：

| 你说的 | 效果 |
|--------|------|
| "把会议改成会谈" | 替换关键词 |
| "去掉最后一句" | 删除末尾内容 |
| "在后面加上请及时回复" | 追加文字 |
| "改成更正式的表达" | LLM 润色重写 |

技术上的精细之处：
- 有一个"文字账本"（TextBuffer）追踪自己打出的所有文字
- 同时监听键盘 Backspace 事件（30秒窗口），实时同步用户手动删除
- 监听鼠标点击事件——如果用户用过鼠标（光标可能位移了），编辑时会自动切换到"行选择模式"，通过 Home→Shift+End→复制 精确拿到当前行的真实内容，而不是依赖可能过时的文字账本

### 4.4 硬件架构 —— 无线 + 双模

硬件采用 **nRF52840（发射端）+ ESP32-S3（接收端）** 的双芯片方案：

- **发射端**：nRF52840，工作电流仅 12-18mA，500mAh 电池续航 **50+ 小时**。集成 PDM 数字 MEMS 麦克风，2 根线直连，近讲拾音。通过 BLE 5.0 传输 PCM 音频数据（16kHz/16bit/mono）。
- **接收端**：ESP32-S3，USB 复合设备——同时呈现 HID 键盘 + CDC 串口 + UAC 麦克风。按一个键即可在"语音打字模式"和"普通 USB 麦克风模式"之间切换。

为什么不用 ESP32 做发射端？ESP32 工作电流 80-150mA，同等电池只能用 3-4 小时，而 nRF52840 专为低功耗 BLE 设计。

设计哲学：设备本身**不连 WiFi，不做语音识别**。音频通过 USB → 电脑，由 Agent 调云端 API。设备换办公室/换 WiFi 零感知，STT 提供商和 API Key 在配置文件里随时切换，无需刷固件。

### 4.5 Claude Code 集成 —— 语音编程

按住 AI 热键（默认右 Shift）说："帮我把所有 print 改成 logging"，松手后 Claude Code 自动读取代码、批量替换，你的编辑器检测到文件变化自动刷新。

支持多轮对话——第一条指令建立上下文，后续指令自动续接，不需要每次重新说明背景。这是一个**真正的语音驱动 AI 编程助手**，不只是"语音转文字再手动粘贴给 ChatGPT"。

## 七、开发进度与路线图

### 已完成 ✅

- 三平台打字（macOS Quartz / Windows SendInput / Linux XTest）
- Push-to-Talk 录音 + VAD 自动唤醒（两种模式）
- 五大 STT 提供商接入（讯飞/阿里云/火山引擎/智谱 AI/OpenAI）
- 四大 LLM 提供商接入（OpenAI/通义千问/豆包/智谱 GLM）
- LLM 语音编辑 + 文字账本 + 光标不确定性检测
- Claude Code AI 编程模式（语音驱动改代码）
- 语音快捷键映射（10+ 常用操作）
- 串口自动识别 + 断线重连
- 开机自启动（macOS LaunchAgent / Windows 注册表 / Linux .desktop）
- ESP32-S3 固件（USB HID + CDC + UAC 复合设备、文字路由、模式切换）
- 剪贴板粘贴模式（微信/钉钉适配）
- 行选择纠错模式（鼠标点击后光标位移的降级方案）
- 性能优化（讯飞发包间隔从 40ms 压到 5ms，长句快 2.6 秒）

### 开发中 🚧

- ESP32-S3 BLE Central：扫描连接 nRF52840 发射端
- BLE GATT Client：接收 PCM 音频数据
- UAC 音频直通：BLE PCM → USB 音频端点

### 待开发 📋

- 快捷键自定义配置文件

### 8.3 情感卖点

- **写的比打的快**：人平均语速 150-200 字/分钟，打字速度 40-60 字/分钟。语音输入天生快 3-5 倍。
- **不打断思路**：灵感来的时候打字的节奏永远跟不上思考的速度。Voice Keyboard 让你可以专注于"想说什么"，而不是"怎么打出来"。
- **沉默的力量**：图书馆、深夜、公共场合——不用大声喊，近讲麦克风让你保持安静的体面。
- **键盘手的解放**：程序员每天敲击键盘数万次，Voice Keyboard 把注释、消息、文档的工作交给语音。

### 8.4 技术硬核点（适合科技媒体/KOL）

- 在 macOS 上用 Quartz Core Graphics 注入 Unicode 事件，彻底绕过 IME（输入法）
- Windows 上 GetForegroundWindow 判断当前窗口，自动在 SendInput 和剪贴板粘贴两种模式间切换
- 文字账本（TextBuffer）+ 键盘 Backspace 监听 + 鼠标点击检测，三路协同确保"语音编辑"知道你要改哪句话
- BLE 5.0 自定义 GATT Service 传输 16kHz PCM 音频，有效带宽占用仅 18%
- ESP32-S3 USB 复合设备（HID + CDC + UAC），一个 USB 口同时是键盘、串口和麦克风

## 九、给市场部的建议

1. **双线并推进**：软件模式做大用户基数（零门槛，填个 API Key 就能用），硬件模式做高价值变现（硬件利润 + 服务免费感知）。

2. **视频 > 文字**：这个产品非常适合短视频展示——"按住说话→文字出现→说话改文字→快捷键触发→AI 编程"五段式 demo，直观冲击力极强。

3. **对比体验**：让用户对比"手动打字 100 字"和"Voice Keyboard 说 100 字"的时间差，量化展示效率提升。

4. **程序员社区先行**：Claude Code 集成是一个非常独特的卖点，"语音驱动 AI 写代码"在程序员群体中有病毒传播潜力。
